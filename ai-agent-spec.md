# AI-наставник внутри Mini App

> **Проект:** FinAssist
> **Дата:** 2026-06-10
> **Приоритет:** High
> **Оценка:** 30 минут

---

## 1. User Story

**Как** пользователь Mini App FinAssist,
**я хочу** задать вопрос AI-наставнику по финансам,
**чтобы** получить короткий, понятный и точный ответ по данным из моего аккаунта.

### Сценарий использования
1. Пользователь открывает Mini App и переходит на экран AI-наставника.
2. В интерфейсе появляется инлайн-чат с полем ввода.
3. Пользователь спрашивает, например, "Почему налоговый фонд ниже нормы?" или "Какие расходы в категории Маркетинг можно сократить?".
4. Backend получает запрос, анализирует данные пользователя и вызывает Claude Opus 4.8 (`claude-opus-4-8`).
5. Сервер возвращает ответ в формате готового текста.
6. Клиент показывает ответ в чате.

### Критерии приёмки
- [ ] AI-наставник отвечает на вопросы по финансам, аналитике и рекомендациям.
- [ ] Ответы формируются за 2-4 предложения.
- [ ] Ответы приходят из backend, без Anthropic вызовов из фронта.
- [ ] Модель используется Claude Opus 4.8 (`claude-opus-4-8`) через Anthropic API.
- [ ] Чат работает как встроенный инлайн-чат внутри Mini App.

## 2. Задачи агента

- Отвечать на вопросы по финансам.
- Объяснять аналитику из дашборда.
- Давать рекомендации по категориям расходов.

## 3. Контекст агента (что он знает)

- Историю транзакций пользователя из Supabase (автоматически синхронизируется из Robokassa и Prodamus).
- Данные по категориям, юрлицам и проектам.
- Правила работы с движением денежных средств (ДДС).
- Правила FinAssist по суммам в копейках, soft delete и датам UTC.

### Источники данных
Агент **только читает** уже синхронизированные транзакции из БД. Синхронизация источников (Robokassa, Prodamus, Tochka), cron-расписание и дедупликация — это платформенный слой, описанный в [mini-app.architector.md](mini-app.architector.md) (раздел «Источники данных и синхронизация»). Здесь это не дублируется.

## 4. Формат работы

- Инлайн-чат прямо в интерфейсе Mini App.
- Мини App реализован на React + Vite + Tailwind.
- Модель: Claude Opus 4.8 (`claude-opus-4-8`) через Anthropic API. Загружается из `config.AI_MENTOR_MODEL` (не хардкодить в сервисе). Отличается от модели классификации транзакций (`config.CLAUDE_MODEL` = `claude-sonnet-4-6`): наставник — качественный диалог, поэтому Opus; классификатор — детерминированный разбор, поэтому Sonnet.
- Ответы: 2-4 предложения, конкретно и по делу.
- Все AI-вызовы только через backend.
- Фронт получает уже готовый текст, не хранит промпты и не выполняет модельные запросы.

## 5. API

### `POST /api/webapp/ai/chat`
**Описание:** отправить вопрос AI-наставнику и получить готовый ответ.
**Авторизация:** требует `telegram_id` / webapp session.
**Запрос:**
```json
{ "question": "Почему налоговый фонд ниже нормы?", "entity_id": "...", "from": "2026-06-01", "to": "2026-06-10", "context": "dashboard" }
```
**Ответ 200:**
```json
{
  "answer": "Налоговый фонд ниже нормы, потому что в этот период поступления от основного направления были ниже привычных, а расходы на маркетинг выросли на 18%. Рекомендуем временно отложить нерегулярные закупки и пересмотреть расходы в категории Маркетинг.",
  "source": "dashboard",
  "timestamp": "2026-06-10T12:00:00Z"
}
```
**Ошибки:**
```json
{ "error": { "code": "invalid_request", "message": "Вопрос не может быть пустым." } }
```

### `POST /api/ai-chat`
**Описание:** основной эндпоинт для AI-наставника Mini App.
**Авторизация:** требует `telegram_id` / webapp session.
**Запрос:**
```json
{ "question": "Какие расходы в категории Маркетинг можно сократить?", "entity_id": "...", "from": "2026-06-01", "to": "2026-06-10", "context": "transactions" }
```
**Ответ 200:**
```json
{
  "answer": "Рекомендуется сократить нерегулярные рекламные закупки и отложить запуск нового проекта до следующего месяца. Это поможет сохранить cash flow и не ухудшить налоговый фонд.",
  "timestamp": "2026-06-10T12:05:00Z"
}
```
**Ошибки:**
```json
{ "error": { "code": "invalid_request", "message": "Вопрос не может быть пустым." } }
```

## 6. Business Logic

### Правила
- Backend выполняет все AI-вычисления.
- Фронтенд получает только готовые тексты, не хранит промпты и не выполняет модельные запросы.
- Многопользовательский доступ определяется через `app_users.telegram_id`.
- Данные сохраняются согласно правилам FinAssist: суммы в копейках, UTC-даты, soft delete.
- Frontend не выполняет Anthropic/Opus/Deepgram запросы.

### Безопасность (специфика агента)
- Все AI-вызовы выполняются только на backend; фронт получает готовый текст, не хранит промпты и не делает модельных запросов.
- Полный текст вопросов/ответов не логируется — только метаинформация (telegram_id, timestamp, длина контекста).
- В промпт передаётся только релевантный срез данных пользователя, не raw-выгрузки.

> Общая безопасность платформы (credentials источников, audit log синхронизаций, rate limiting, SSL/TLS) описана в [mini-app.architector.md](mini-app.architector.md) (раздел «Безопасность и защита данных»).

### Интеграции
- **Anthropic Claude Opus 4.8 (`claude-opus-4-8`)**: генерирует ответы по вопросу и контексту (модель из `config.AI_MENTOR_MODEL`).
- **Supabase / postgres.js**: предоставляет историю транзакций, категории, направления и данные по ДДС.
- **services/analytics.ts**: готовит contextual snippets для модели и формирует выводы.

> Данные в БД поставляет платформенная синхронизация (Robokassa/Prodamus/Tochka) — см. [mini-app.architector.md](mini-app.architector.md). Агент эти интеграции не вызывает напрямую.

## 7. Edge Cases

| # | Ситуация | Поведение системы |
|---|----------|------------------|
| 1 | Вопрос пустой | Вернуть 400 + сообщение "Введите ваш вопрос" |
| 2 | Данных недостаточно для ответа | Ответить "Недостаточно данных" и предложить изменить период или загрузить выписку |
| 3 | Запрос слишком общий | Дать краткий общий ответ и предложить уточнить категорию или период |
| 4 | Backend AI временно недоступен | Вернуть сообщение "AI-сервис недоступен, повторите позже" |
| 5 | Пользователь спрашивает о политике/нефинансовом | Ответить, что агент отвечает только на вопросы по финансам и аналитике |

> Edge cases по синхронизации источников и безопасности credentials — в [mini-app.architector.md](mini-app.architector.md).

> Инфраструктурные файлы (интеграции источников, env-переменные credentials, cron) описаны в [mini-app.architector.md](mini-app.architector.md), не дублируются здесь.

### План работы
1. Создать компонент `AIChatWidget` (инлайн-чат в углу экрана).
2. Реализовать API endpoint `POST /api/ai-chat` (принимает вопрос пользователя и контекст).
3. Подключить Anthropic API для Claude Opus 4.8 (`claude-opus-4-8`, из `config.AI_MENTOR_MODEL`) на backend.
4. Передавать в промпт агента релевантный срез данных пользователя из БД (история транзакций, категории, ДДС).
5. Реализовать AI-специфичные security checks: не логировать текст вопросов/ответов, передавать в промпт только нужный срез.

> Предпосылка: данные в БД уже синхронизированы платформенным слоем (см. [mini-app.architector.md](mini-app.architector.md)). Этот план не включает настройку синхронизации источников.

## Итог
AI-наставник в Mini App — это компактный backend-driven чат для финансовых вопросов, аналитики и рекомендаций. Он использует Claude Opus 4.8 (`claude-opus-4-8`, из `config.AI_MENTOR_MODEL`), читает актуальные данные пользователя из БД (которые туда кладёт платформенная синхронизация — см. [mini-app.architector.md](mini-app.architector.md)) и отвечает коротко, по делу, без AI-запросов из браузера. Агент — потребитель данных, а не владелец интеграций.
