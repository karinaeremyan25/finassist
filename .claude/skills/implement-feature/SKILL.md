Ты получил спецификацию фичи для FinAssist. Выполни пошагово:

1. **Анализ** — прочитай CLAUDE.md, SPEC.md и описание задачи. Определи затронутые модули (handlers, services, repositories, db). Задай уточняющие вопросы ПЕРЕД началом, если спецификация неполная.

2. **База данных** (если нужны изменения схемы) — делегируй субагенту `database-architect`:
   - Новые таблицы или поля → новая миграция в `db/migrations/`
   - Напоминание: все суммы BIGINT копейки, UUID, soft delete, без RLS

3. **Репозитории** — делегируй субагенту `backend-engineer`:
   - Функции в `src/db/repositories/` для работы с новыми/изменёнными таблицами
   - Только параметризованные запросы postgres.js
   - Без фильтрации по роли (все три роли имеют равный доступ)

4. **Сервисы и handlers** — делегируй субагенту `backend-engineer`:
   - Бизнес-логика в `src/services/`
   - grammY handler в `src/bot/handlers/`
   - Inline keyboard если нужен диалог
   - Zod-валидация, retry, pino-логирование

5. **AI-логика** (если фича затрагивает классификацию, парсинг, промпты) — делегируй субагенту `ai-agent-architect`:
   - Изменения в `classifier.ts`, `claude.ts` или `parser/`
   - Проверить confidence thresholds и fallback

6. **Деплой-конфиг** (если новые env-переменные или cron) — делегируй субагенту `devops-engineer`

7. **Проверка** — делегируй субагенту `qa-reviewer`:
   - Безопасность: auth middleware, SQL-инъекции
   - Роли: нет фильтрации по роли
   - Деньги: нет float
   - Edge cases из задачи

8. **Отчёт** — кратко: какие файлы созданы/изменены, что теперь умеет бот.

Используй plan mode перед началом реализации для согласования архитектуры.
