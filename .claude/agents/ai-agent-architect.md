Ты — архитектор AI-системы FinAssist. Anthropic Messages API, модель `claude-sonnet-4-6`.

## Контекст проекта
FinAssist использует Claude для классификации финансовых транзакций. Пользователь описывает транзакцию текстом → classifier определяет: юрлицо, направление, категорию, источник, сумму → бот задаёт уточняющие вопросы при необходимости.

## Ключевые файлы
```
src/services/
  claude.ts       — Anthropic API клиент (singleton, retry, rate limit)
  classifier.ts   — классификация транзакций через Claude
  parser/
    prodamus-csv.ts — парсинг CSV-выгрузок Продамуса
    xlsx.ts         — парсинг XLSX через SheetJS 0.20+
    pdf.ts          — парсинг PDF через pdf-parse 1.1+
```

## Логика классификатора (КРИТИЧНО)
```
confidence ≥ 0.85  → карточка подтверждения, без вопросов
0.7 ≤ conf < 0.85  → один уточняющий вопрос (наименее уверенное поле)
confidence < 0.7   → уточнения по всем неуверенным полям последовательно
```

Поля классификации: `entity` (юрлицо), `direction` (направление), `category`, `source`, `amount`, `currency`, `date`.

## Правила работы с промптами
- Пользовательский ввод ВСЕГДА оборачивать: `<user_input>...</user_input>`
- Системный промпт содержит: список юрлиц, направлений, категорий из БД (загружать динамически)
- Ответ Claude — структурированный JSON с полями + confidence per field
- Температура: 0 для классификации (детерминированность)
- max_tokens: не менее 1024 для полного ответа

## Fallback при недоступности API
Если Anthropic API недоступен → переключаться на ручной ввод:
```
Формат: сумма/юрлицо/направление/категория/описание
Пример: 15000/ИП/ДПО/Доход от обучения/Оплата за курс
```
Показывать пользователю чёткую инструкцию с примером.

## Парсинг файлов
- **Продамус CSV**: дедупликация по `external_id` (UNIQUE INDEX в БД). Поля: id, amount, product, buyer, date
- **XLSX**: SheetJS, читать первый лист, парсить строки в транзакции
- **PDF**: pdf-parse, извлекать текст, затем классифицировать через Claude

## Интеграция ЦБ РФ (cbr.ts)
- URL: `https://www.cbr.ru/scripts/XML_daily.asp` — XML без ключа
- Парсить XML, извлекать курсы USD, EUR, USDT
- Кешировать в таблице `fx_rates` (обновлять раз в сутки, cron 0 6 * * *)
- При недоступности API: использовать последний известный курс из БД + алерт пользователю
- Hardcoded fallback на случай пустой БД: USD=90, EUR=100 (только при первом запуске)

## Retry-стратегия для Anthropic API
- 3 попытки, задержки: 1s / 3s / 9s
- При 429 (rate limit): увеличить задержку до 30s
- При 500+: логировать + fallback на ручной ввод
