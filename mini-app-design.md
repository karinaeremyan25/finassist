# Дизайн Mini App — «Психология Здоровья»

> **Проект:** FinAssist · Telegram Mini App
> **Дата:** 2026-06-10
> **Тема:** глубокий морской дашборд (deep ocean / petrol teal)
> **Статус:** дизайн-сборка (design system + макет Dashboard)
> **Источник вдохновения:** референс-лейаут финансового приложения (deep-teal). Бренд **не** «Aksari» — это **«Психология Здоровья»**.

Сборка выполнена с участием Claude-скиллов `ui-ux-pro-max` + `design-system` + `ui-styling`.
Палитра снята пипеткой с референса (см. раздел 2) и приведена к доступным морским токенам.

## 1. Концепция

Спокойный, «глубоководный» финансовый дашборд: тёмно-бирюзовый фон как толща воды, светящиеся аква-акценты как блики на поверхности. Тон — клинически чистый, премиальный, без игривости. Бренд **«Психология Здоровья»** (направление: Курс ДПО «Психология здоровья»).

| Принцип | Решение |
|---------|---------|
| **Стиль** | Dark Mode (OLED), высокий контраст, WCAG AA/AAA |
| **Настроение** | море, глубина, спокойствие, доверие, точность |
| **Шрифт** | Inter (заголовки + текст), табличные цифры для сумм |
| **Платформа** | Telegram Mini App, mobile-first 360–420px |
| **Анти-паттерны** | ❌ фиолетово-розовые AI-градиенты · ❌ эмодзи вместо иконок · ❌ игривые формы · ❌ непрозрачные цифры |

### 2.4 Контраст (проверено)

| Пара | Ratio | Норма |
|------|-------|-------|
| `--text` на `--bg-deep` | ~13:1 | AAA ✓ |
| `--text-muted` на `--bg-deep` | ~4.8:1 | AA ✓ |
| `--accent-ink` на `--accent` | ~9:1 | AAA ✓ |

## 3. Типографика

**Шрифт:** Inter (300/400/500/600/700). Для всех сумм — табличные цифры.

## 6. Donut-диаграмма (замена блока «Профит 70%»)

> Главный экран сохраняет **круговую (кольцевую) диаграмму** в стиле референса: кольцо слева, легенда справа, в центре — крупный «профит %».
>
> **Математически чистое кольцо (part-to-whole):** целое = **Выручка** (100%). Кольцо показывает, **куда распределяется выручка** — на непересекающиеся доли: **Расходы + Налоги + Фонд «Благодарность» + Фонд «Кредиты» + Прибыль (чистый доход)**. Доли в сумме = 100%, центр = доля Прибыли. Это убирает прежнее пересечение «Выручка ⊇ Доходы». Все цвета — морской спектр.

### 6.1 Параметры (по правилам `ui-ux-pro-max`)

| Параметр | Значение |
|----------|----------|
| Тип | **Donut** (кольцо), не pie — центр под метрику |
| Целое | **Выручка** за период (= 100%) |
| Категорий | **5** непересекающихся долей: Расходы, Прибыль, Фонд «Кредиты», Налоги, Фонд «Благодарность» |
| Толщина кольца | 18–22px, скруглённые концы сегментов |
| Старт | крупнейший сегмент на **12 часов**, по часовой |
| Метки | **всегда %** + абсолют в легенде |
| Центр | «ПРИБЫЛЬ 35%» (доля прибыли = margin %) — крупно, `--text` |
| Подпись | «от выручки ₽ 2 840 000 · сумма долей = 100%» |
| Порядок | по убыванию доли, **крупнейший сегмент на 12 часов** |
| Анимация | прорисовка 600ms `ease-out`, отключать при reduced-motion |
| Взаимодействие | tap по сегменту → tooltip с точной суммой (тач-таргет ≥44pt) |

### 6.2 Цвета сегментов (морской спектр), порядок по убыванию доли

| Сегмент | Токен | Hex | Демо-доля |
|---------|-------|-----|-----------|
| **Расходы** | `--chart-expense` | `#FB7A6E` (коралл) | 1 240K · 44% |
| **Прибыль** | `--chart-profit` | `#34D399` (морская зелень) | 1 004K · 35% |
| **Фонд «Кредиты»** | `--chart-fund-cred` | `#FBBF24` (янтарь) | 284K · 10% |
| **Налоги** | `--chart-tax` | `#94A3B8` (нейтральный сланец) | 170K · 6% |
| **Фонд «Благодарность»** | `--chart-fund-grat` | `#38BDF8` (лагуна) | 142K · 5% |

5 максимально различимых оттенков (коралл / зелень / янтарь / сланец / синий) — соседние сегменты кольца не сливаются. Налоги — нейтральный серый (обязательство), Прибыль — позитивная морская зелень.

### 6.3 Обязательный a11y-fallback

Pie/donut проваливает WCAG для дальтоников (опора на цвет). Поэтому **легенда под кольцом обязательна** и служит таблицей данных:
- каждая строка: цветовой маркер + название + сумма + %;
- маркеры отличаются и формой (●/◆/■), не только цветом;
- кольцу даётся `role="img"` + `aria-label` с кратким выводом, плюс скрытая таблица значений для скринридеров.

### 6.4 Маппинг на API

Данные берём из `GET /api/analytics/summary` (суммы в **копейках**, форматируем через `utils/money.ts`):

```ts
// summary: { totalIncome (= Выручка), totalExpense, fundStatus, ... }
// fundStatus расширяется кастомными фондами проекта:
// fundStatus = { taxFund, reserveFund, gratitudeFund, creditFund }
const revenue = summary.totalIncome;                 // целое (100%)
const profit  = revenue
  - summary.totalExpense
  - summary.fundStatus.taxFund
  - summary.fundStatus.gratitudeFund
  - summary.fundStatus.creditFund;                   // остаток = чистая прибыль

const segments = [
  { key: 'expense', label: 'Расходы',             value: summary.totalExpense,             color: 'var(--chart-expense)'   },
  { key: 'profit',  label: 'Прибыль',             value: profit,                           color: 'var(--chart-profit)'    },
  { key: 'cred',    label: 'Фонд «Кредиты»',       value: summary.fundStatus.creditFund,    color: 'var(--chart-fund-cred)' },
  { key: 'tax',     label: 'Налоги',              value: summary.fundStatus.taxFund,       color: 'var(--chart-tax)'       },
  { key: 'grat',    label: 'Фонд «Благодарность»', value: summary.fundStatus.gratitudeFund, color: 'var(--chart-fund-grat)' },
].sort((a, b) => b.value - a.value);                 // крупнейший на 12 часов

// центр кольца = доля прибыли (margin)
const marginPct = Math.round((profit / revenue) * 100); // «ПРИБЫЛЬ 35%»
```

> Все доли — непересекающиеся части одной Выручки, поэтому Σ долей = 100% (математически чисто).
> Фонды «Благодарность» и «Кредиты» — кастомные фонды проекта; бэкенд отдаёт их в `fundStatus` (`gratitudeFund`, `creditFund`) из таблицы фондов; в БД суммы — копейки. «Прибыль» — вычисляемый остаток, отдельно в БД не хранится.

> ⚠️ Корректность данных: для строгого part-to-whole сегменты кольца должны быть **непересекающимися** долями одного целого. Если «Выручка» и «Доходы» пересекаются по смыслу — рекомендуется собирать кольцо из `categoryBreakdown` (непересекающиеся категории), а тройку Выручка/Доходы/Расходы оставить как сравнительные KPI. Текущая сборка следует прямому ТЗ (3 названных сегмента) + показывает margin в центре.

### 6.5 Реализация (Recharts)

```tsx
import { PieChart, Pie, Cell, ResponsiveContainer, Tooltip } from 'recharts';

// порядок соответствует segments после сортировки по убыванию доли
const COLORS = ['#FB7A6E', '#34D399', '#FBBF24', '#94A3B8', '#38BDF8'];

<div role="img" aria-label={`Распределение за период: Выручка ${pct(0)}%, Доходы ${pct(1)}%, Расходы ${pct(2)}%`}>
  <ResponsiveContainer width="100%" height={200}>
    <PieChart>
      <Pie data={segments} dataKey="value" nameKey="label"
           innerRadius={64} outerRadius={88} paddingAngle={2}
           startAngle={90} endAngle={-270} cornerRadius={8} stroke="none">
        {segments.map((s, i) => <Cell key={s.key} fill={COLORS[i]} />)}
      </Pie>
      <Tooltip formatter={(v) => rubles(BigInt(v))} />
    </PieChart>
  </ResponsiveContainer>
  {/* центр кольца — абсолютным позиционированием поверх */}
  <div className="donut-center"><b className="num">{marginPct}%</b><span>Прибыль</span></div>
</div>
```

## 7. Компоненты

### 7.1 Шапка (Header)
- Фон `--grad-header`, высота под `safe-area-inset-top`.
- Слева — бренд «ПСИХОЛОГИЯ ЗДОРОВЬЯ» (caption, UPPERCASE, letter-spacing .06em, `--accent-soft`).
- Справа — аватар-кружок с инициалом и иконка настроек (Lucide `settings`).

### 7.2 Карточка баланса
- Фон `--grad-balance`, радиус `--r-lg`, тень `--elev-2`.
- «ОБЩИЙ БАЛАНС» (label, muted) → сумма (display, `.num`) → дельта со стрелкой (`--income`/`--expense`).
- Тап по карточке → переключение периода (правило `KPI кликабельны`).

### 7.3 KPI-карточки (Доходы / Расходы)
- Две в ряд, `--surface-2`, радиус `--r-md`.
- Иконка в кружке (`arrow-down-left` доход / `arrow-up-right` расход), сумма `.num`, дельта %.
- Цвет суммы наследует `--income` / `--expense`; знак обязателен.

### 7.4 Список операций
- Строка: иконка категории (квадрат `--r-sm`, `--surface-3`) · название · мета (направление · дата) · сумма (`.num`, цвет по знаку).
- Разделители — `--border`. Заголовок секции + ссылка «Все →».
- Виртуализация при 50+ строках.

### 7.5 Карточки инсайтов
- Из `GET /api/analytics/insights`: иконка-ромб `--accent`, title (h2-стиль), text (body, muted).
- Готовый текст с бэкенда (AI не вызывается из фронта).

### 7.6 Нижняя навигация (Bottom Tab Bar)
- 4 пункта (≤5): **Главная · Отчёты · Счета · Чат**.
- Фон `--surface-nav` + blur, верхняя граница `--border`, `safe-area-inset-bottom`.
- Активная вкладка: иконка+подпись `--accent`, лёгкое свечение `--glow-accent`. Иконка **и** подпись обязательны.
- AI-наставник = вкладка «Чат» (виджет `AIChatWidget`).

## 10. Технические заметки

- **Стек:** React + Vite + Tailwind (подтверждён в SESSION_HANDOFF). Токены раздела 2 → `:root` + `tailwind.config` (extend colors).
- **Telegram:** учитывать `themeParams`, но визуально форсим собственную тёмно-морскую тему (не наследуем светлую тему клиента).
- **Деньги:** все суммы приходят в копейках (BIGINT) → форматирование только через `utils/money.ts` (`rubles()`), отображение «1 500,50 ₽».
- **Даты:** приходят UTC → формат в МСК (UTC+3) на уровне отображения.
- **Backend-driven:** фронт не вызывает Anthropic/Opus/Deepgram; графики строятся из chart-ready payload и `summary`.
- **Иконки:** Lucide (один набор, stroke 1.5–2px), без эмодзи.
