# Feature: AI-агент аналитики в Telegram Mini App

> **Проект:** FinAssist
> **Дата:** 2026-06-10
> **Приоритет:** High
> **Оценка:** 30 минут

---

## 1. User Story

**Как** пользователь FinAssist,
**я хочу** получать готовые аналитические выводы и вопросы через Mini App,
**чтобы** не дублировать бизнес-логику бота и быстро видеть результат без прямых AI-запросов из браузера.

### Сценарий использования
1. Пользователь открывает Mini App из Telegram внутри своего канала или чата.
2. Mini App инициализируется через `POST /api/webapp/session` и получает профиль пользователя.
3. Пользователь выбирает период, юрлицо или направление.
4. Клиент делает запрос к backend за аналитикой и инсайтами.
5. Backend возвращает готовые KPI, chart-ready payload и текстовые аналитические карточки.
6. Mini App показывает дашборд, графики и инсайты, без вызова Anthropic из фронтенда.
7. Если пользователь хочет посмотреть список транзакций или участников — Mini App загружает их через API.

### Критерии приёмки
- [ ] Mini App открывается и проходит авторизацию через Telegram session.
- [ ] Все AI-вызовы выполняются на backend, фронтенд получает только готовые данные и тексты.
- [ ] В дашборде отображаются KPI, графики и карточки инсайтов.
- [ ] Фронтенд не требует Anthropic, Opus или Deepgram API.
- [ ] Многопользовательский доступ работает через существующий `telegram_id` в `app_users`.
- [ ] Ошибки API обрабатываются корректно: loading / empty / error.

## 3. Изменения в API

#### `POST /api/webapp/session`
**Описание:** инициализация Mini App, проверка Telegram initData и подготовка метаданных.
**Авторизация:** требует `initData` от Telegram.
**Запрос:**
```json
{ "initData": "<telegram_init_data>" }
```
**Ответ 200:**
```json
{
  "user": { "telegram_id": 12345, "name": "Карина", "role": "owner" },
  "entities": [ { "id": "...", "name": "ИП Карина Еремян" } ],
  "availableDirections": [ { "id": "...", "name": "Курс ДПО" } ],
  "defaultPeriod": { "from": "2026-06-01", "to": "2026-06-10" },
  "features": ["analytics", "transactions", "users"]
}
```
**Ответ ошибки:**
```json
{ "error": { "code": "unauthorized", "message": "Сессия не распознана" } }
```

#### `GET /api/analytics/summary`
**Описание:** агрегированная сводка по выбранному периоду.
**Авторизация:** требует `telegram_id`.
**Запрос:** `?entity_id=&from=&to=&direction_id=`.
**Ответ 200:**
```json
{
  "totalIncome": 12000000,
  "totalExpense": 4500000,
  "balance": 7500000,
  "fundStatus": {
    "taxFund": 900000,
    "reserveFund": 300000,
    "gratitudeFund": 142000,
    "creditFund": 284000,
    "profitFund": 1200000
  },
  "categoryBreakdown": [
    { "category": "Маркетинг", "amount": 1500000 },
    { "category": "Зарплата", "amount": 2000000 }
  ]
}
```

> `fundStatus` (копейки): `taxFund`/`reserveFund` — из балансов фондов; `gratitudeFund`/`creditFund` — кастомные фонды проекта, считаются из транзакций за период (loan = `getLoanExpenseMetrics`, благодарность = `getGratitudeFundMetrics`). Нужны для donut Dashboard (`mini-app-design.md` §6.4).

#### `GET /api/analytics/charts`
**Описание:** подготовка данных для графиков.
**Авторизация:** требует `telegram_id`.
**Запрос:** `?entity_id=&from=&to=&group_by=day|week|month`.
**Ответ 200:**
```json
{
  "labels": ["01.06", "02.06"],
  "series": [
    { "name": "Доход", "data": [500000, 600000] },
    { "name": "Расход", "data": [200000, 250000] }
  ],
  "type": "line"
}
```

#### `GET /api/analytics/insights`
**Описание:** готовые текстовые аналитические выводы.
**Авторизация:** требует `telegram_id`.
**Запрос:** `?entity_id=&from=&to=`.
**Ответ 200:**
```json
{
  "insights": [
    { "title": "Рост выручки", "text": "Выручка за неделю выросла на 12% по сравнению с предыдущей." },
    { "title": "Фонд налога", "text": "Налоговый фонд сформирован на 74%, рекомендуем отложить дополнительные 15000 ₽." }
  ]
}
```

#### `GET /api/analytics/transactions`
**Описание:** список транзакций для выбранного периода.
**Авторизация:** требует `telegram_id`.
**Запрос:** `?entity_id=&from=&to=&limit=&offset=`.
**Ответ 200:**
```json
{
  "transactions": [
    { "id": "...", "date": "2026-06-08T12:00:00Z", "description": "Оплата курса", "amount": 250000, "direction": "Курс ДПО", "category": "Доход" }
  ],
  "total": 42
}
```

#### `GET /api/webapp/users`
**Описание:** список пользователей с доступом.
**Авторизация:** требует `telegram_id`.
**Ответ 200:**
```json
{
  "users": [
    { "telegram_id": 12345, "name": "Карина", "role": "owner", "last_seen": "2026-06-10T09:00:00Z" }
  ]
}
```

### Изменения в существующих эндпоинтах
- `GET /api/users/me`: расширяется метаданными для Mini App.
- `POST /api/transactions/import`: может логировать источник `source: "webapp"`, без изменения бизнес-логики.

## 5. Business Logic

### Правила
- AI-логика выполняется только на backend.
- Фронтенд получает только готовые тексты, графики и числовые агрегаты.
- Многопользовательский доступ основывается на `app_users.telegram_id`.
- Данные сохраняются и выводятся согласно правилам FinAssist: суммы в копейках, UTC-даты, soft delete.
- Frontend не выполняет Anthropic/Opus/Deepgram запросы.

### Интеграции
- **Сервис:** `services/analytics.ts`
  - Что отправляем: `entity_id`, `direction_id`, `from`, `to`, `telegram_id`.
  - Что получаем: агрегаты, карточки KPI, графики, инсайты.
  - При ошибке: логируем и возвращаем `500` с понятным сообщением.

- **Сервис:** `services/webapp.ts`
  - Что отправляем: `initData`.
  - Что получаем: профиль пользователя, разрешённые сущности, период по умолчанию.

- **Сервис:** `services/claude.ts` / `services/classifier.ts`
  - Используются только для backend AI, если нужны дополнительные инсайты или классификация.

## 6. Edge Cases

| # | Ситуация | Поведение системы |
|---|----------|-------------------|
| 1 | Telegram initData не валиден | Ответ 401, сообщение "Сессия не распознана" |
| 2 | Нет транзакций за период | Показать пустое состояние с подсказкой "Добавьте выписку или выберите другой период" |
| 3 | Частичные данные (нет фонда или направления) | Показать доступные блоки, скрыть пустые графики |
| 4 | Неверный фильтр | Ответ 400 + сообщение "Неверный фильтр" |
| 5 | API аналитики недоступен | Показать error-state с кнопкой "Попробовать снова" |
| 6 | Backend AI сервис временно недоступен | Показать карточку "Инсайты недоступны" и вернуть основной дашборд |

## Итог
AI-агент внутри Mini App останется backend-driven: готовые тексты, уменьшенная фронтенд-логика, никаких прямых Anthropic/Opus вызовов в браузере. Это решение позволяет быстро добавить платформу аналитики и многопользовательский доступ, сохранив текущую архитектуру FinAssist.
