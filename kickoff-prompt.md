# 4-21 | Kickoff Prompt — Запуск автономной сборки

> **Тип:** Instruction
> **Когда использовать:** После того как SPEC.md написана, Claude Code Setup сгенерирован, все файлы (CLAUDE.md, субагенты, rules, skills) готовы к созданию
> **Результат:** Claude Code создаёт все конфигурационные файлы и начинает реализацию проекта по спецификации

---

## Философия этого документа

Этот файл — **готовый промпт для Claude Code**. Вы не пишете его сами — копируете и используете.

**Принцип работы:**
- Вы — архитектор и коммуникатор
- Claude Code — исполнитель: читает промпт, делает работу
- Ваша задача — подготовить контекст (SPEC.md, настройки) и скопировать правильный промпт

**Workflow:**
```
Вы → копируете промпт → вставляете в Claude Code (VS Code)
       ↓
       SPEC.md + пакет настроек уже лежат в проекте
       ↓
Claude Code → выполняет автономную сборку по промпту
       ↓
Вы → принимаете/правите результат → тестируете
```

Промпт уже отлажен. Не нужно его улучшать с первого раза — просто используйте.

---

## Что это такое

Это **финальный промпт**, который ты копируешь в Claude Code для запуска полной автономной сборки проекта. Claude Code читает SPEC.md, создаёт структуру проекта, затем реализует фичи в правильном порядке через субагентов.

**Порядок сборки:**
1. Database (схема, миграции, RLS) → database-architect
2. Backend (API routes, бизнес-логика) → backend-engineer
3. Frontend (компоненты, страницы) → frontend-developer
4. QA (тестирование, review) → qa-reviewer

- Каждый шаг завершай коммитом с осмысленным сообщением

- Если встречаешь неоднозначность в спецификации — используй лучшую практику, не останавливайся
- Все строки — в TypeScript, никакого JavaScript

## Чего ожидать после запуска

### Первые 2-3 минуты
Claude Code создаёт файлы конфигурации и инициализирует проект. Ты увидишь:
- Создание CLAUDE.md, субагентов, rules
- `npx create-next-app@latest`
- Установку зависимостей (`npm install`)
- Первый коммит

### 5-15 минут
Создание схемы БД через Supabase MCP. Claude Code:
- Выполняет SQL-миграции
- Настраивает RLS-политики
- Проверяет что таблицы созданы корректно

### 15-40 минут
Реализация бэкенда и фронтенда. Это самая длинная фаза:
- Создаёт файлы в `src/app/` и `src/components/`
- Server Actions в `src/app/actions/`
- Типы в `src/types/`

### 40-50 минут
QA-фаза. Claude Code проверяет собственный код.

## Как мониторить прогресс

### В терминале Claude Code
Смотри на:
- Какой файл сейчас создаётся/редактируется
- Git-коммиты (каждый шаг = коммит)
- Ошибки TypeScript (если Claude Code запускает `tsc`)

### Проверяй через git
```bash
# Сколько коммитов сделано
git log --oneline

# Какие файлы изменены
git diff --stat HEAD~1

# Структура проекта
find src -type f | head -50
```

### Проверяй через Supabase Dashboard
- Зайди в Table Editor — таблицы должны появиться
- Проверь Authentication → Policies — RLS должен быть включён
- Проверь SQL Editor — можешь выполнить тестовые запросы

## Когда нужно вмешаться

### Красные флаги — ОСТАНОВИ и исправь
| Ситуация | Что делать |
|----------|------------|
| Claude Code зациклился на одной ошибке (3+ попытки) | Останови, прочитай ошибку, дай конкретную инструкцию |
| Создаёт файлы не в тех папках | Останови, укажи правильную структуру |
| Устанавливает пакеты не из стека (например, Prisma вместо Supabase) | Останови, напомни про стек из CLAUDE.md |
| Пишет JavaScript вместо TypeScript | Останови, напомни: "Only TypeScript, .ts/.tsx files" |
| Игнорирует RLS и пишет запросы без auth | Останови, укажи на security rule |

### Жёлтые флаги — наблюдай
| Ситуация | Что делать |
|----------|------------|
| Создаёт компоненты в другом стиле чем ожидал | Подожди до конца шага, потом скорректируй |
| Пропустил одну из фич | Напомни после текущего шага |
| Коммит-сообщения неинформативные | Некритично, можно исправить позже |

### Зелёные сигналы — всё ок
- Git-коммиты появляются регулярно
- Файлы создаются в правильных папках
- TypeScript компилируется без ошибок
- Таблицы появились в Supabase
